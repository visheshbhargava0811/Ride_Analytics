# Ride_Analytics
An end-to-end ride analytics platform combining real-time and batch ingestion on Azure. A FastAPI event producer streams ride-booking events into Azure Event Hub, while Azure Data Factory loads 2,000 rows of historical mapping data into ADLS Gen2. Databricks processes both through a Bronze/Silver/Gold medallion lakehouse built with Spark Declarative Pipelines — 6 metadata-driven, Jinja-templated lookup joins and a 3-minute event-time watermark handle late-arriving records, consolidating into a One Big Table. The gold layer serves a Star Schema with 6 dimension tables (Passenger, Driver, Vehicle, Payment, Booking, Location) and 1 fact table, applying SCD Type 2 through a Spark Structured Streaming CDC flow to track historical city/region changes, with SCD Type 1 elsewhere.
